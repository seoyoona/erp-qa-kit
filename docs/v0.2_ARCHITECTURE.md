> **Status:** Planning only. No code is written and no v0.1 file is modified by this document. This is the *module/engine architecture* reference (the doc named in `docs/v0.2_PLAN.md` §"Sibling Documents"). It explains **how v0.2 extends the v0.1 package layout cleanly** — additive packages and modules, the PROPOSE/VERIFY/APPROVE separation, and the qa-context write-containment guard.

> **Sibling docs (referenced, not duplicated):** functional spec + mismatch taxonomy → `docs/v0.2_SPEC.md`; extractor adapter detail → `docs/v0.2_EXTRACTOR_STRATEGY.md`; policy/memory schemas → `docs/v0.2_PROJECT_POLICY_SPEC.md` and `docs/v0.2_PROJECT_MEMORY_SPEC.md`; hard guarantees → `docs/v0.2_CONSTRAINTS.md`; done definition → `docs/v0.2_ACCEPTANCE_CRITERIA.md`; overview → `docs/v0.2_PLAN.md`.

## 1. Overview — how v0.2 extends v0.1

v0.2 is **purely additive**. Every v0.1 file in the canonical layout stays exactly as it is:

```
erpqa/
  cli.py                      (v0.1 — build_parser/main pattern reused, EXTENDED only by adding subcommands)
  core/  paths.py constants.py errors.py yaml_io.py validation.py scaffold.py sql_safety.py
  adapters/  intake.py
  generators/  sql.py handoff.py
  reporters/  report.py
```

v0.2 adds new *modules* into the existing packages and one new *sub-package* (`erpqa/adapters/extractors/`). Nothing is rewritten; v0.1's 19 tests remain a green gate.

The architecture preserves three invariants from v0.1, restated for v0.2:

| Invariant | v0.1 mechanism | v0.2 extension |
|-----------|----------------|----------------|
| **PROPOSE / VERIFY / APPROVE** separation | LLM/skill proposes rules; deterministic `sql.py`/`validation.py` verify; human approves. | LLMs/skills propose contracts & fixes; deterministic **extractors** + **`compare.py`** verify (no LLM in the verify path); human confirms `needs_human_confirmation` and approves handoffs. |
| **Write containment** | ALL writes go through `paths.py` helpers; `assert_within_qa_context` refuses writes outside `qa-context/`. | Every new writer (contracts, reports, handoff) MUST route through `paths.py`. New `module_paths.py` composes module-relative paths and re-asserts containment — it never bypasses the guard. |
| **Hard guarantees** | Never modify target code; never execute DB; SELECT-only SQL. | Unchanged. Extractors are read-only and offline; `procedure_sql_adapter` parses text but **never executes**; comparison is deterministic and offline. |

```
                v0.1 (shipped, untouched)                      v0.2 (additive)
   ┌───────────────────────────────────────┐   ┌──────────────────────────────────────────────┐
   │ cli.build_parser/main                  │   │ + policy-init/module-init/extract-*/           │
   │ core.paths (containment guard)         │◄──┤   compare-contract/module-report/-handoff/     │
   │ core.constants / errors / yaml_io      │   │   module-audit  (slot into build_parser)       │
   │ core.scaffold / validation / sql_safety│   │ + core.policy / memory / module_paths /        │
   │ adapters.intake                        │   │   contracts / compare                          │
   │ generators.sql / handoff               │   │ + adapters.extractors.* (registry + adapters)  │
   │ reporters.report                       │   │ + generators.frontend_handoff                  │
   └───────────────────────────────────────┘   │ + reporters.module_report                      │
                                                └──────────────────────────────────────────────┘
```

## 2. Module Map (extended)

All paths are relative to the repo root. **NEW** = added by v0.2; everything else is v0.1 (unchanged).

| Module | Status | MUST do | MUST NOT do |
|--------|--------|---------|-------------|
| `erpqa/cli.py` | v0.1, EXTENDED | Register new subcommands via the existing `commands` dict pattern in `build_parser()`; dispatch `args.func`; keep `ErpqaError`/`UsageError` exit codes; generic → exit 3. | Change v0.1 subcommand behavior; add a second positional arg shape; introduce a new dispatch mechanism. |
| `erpqa/core/policy.py` | **NEW** | Load + validate `project_policy.yaml` and `module_policy.yaml`; compute **merge precedence** (module overrides project); expose typed accessors (`frontend_override_allowed`, `validation_order`, `defer_until_frontend_verified`, `allowed_file_formats`). | Write files; make network calls; silently default when `project_policy.yaml` is missing (must halt or draft-with-confirmation). |
| `erpqa/core/memory.py` | **NEW** | Load `project_memory.md` (+ `module_memory.md`); return content **and a "memory was read" provenance record** for reports/handoffs; apply module-refines-project ordering. | Modify memory files implicitly; treat memory as machine truth (it is human-readable context). |
| `erpqa/core/module_paths.py` | **NEW** | Resolve `qa-context/modules/{MODULE}/...` paths **through** `paths.assert_within_qa_context`; provide helpers (`module_dir`, `contract_path`, `module_report_path`, `module_handoff_path`). | Construct or write any path that bypasses the containment guard; escape `qa-context/`. |
| `erpqa/core/contracts.py` | **NEW** | Define the four contract schemas (screen/frontend/backend/procedure) sharing a **provenance mixin** (`source`/`confidence`/`needs_human_confirmation`); validate + (de)serialize via `yaml_io`. | Embed extraction logic; embed comparison logic; allow `confidence: high` on image/OCR-derived fields. |
| `erpqa/core/compare.py` | **NEW** | Deterministic mismatch engine: align screen (source of truth) + backend evidence vs frontend; emit typed mismatches with severity; propagate `needs_human_confirmation`; enforce `frontend_override_allowed=false`. | Call any LLM; auto-resolve conflicts; mutate contracts; reach the network or DB. |
| `erpqa/adapters/extractors/registry.py` | **NEW** | Map `extension → adapter`, gated by policy `allowed_file_formats` **and** `adapter.available()`; expose `select(path, policy)`. | Register future adapters (docling/unstructured); register optional adapters that are not importable. |
| `erpqa/adapters/extractors/<adapter>.py` | **NEW** | One module per REQUIRED adapter (xlsx text, xlsx image-manifest, csv, markdown, json/yaml, frontend-static, backend-static, procedure-sql); each implements the common `Extractor` interface; read-only. | Execute DB/SQL; perform OCR (image-manifest lists positions only); modify source files. |
| `erpqa/generators/frontend_handoff.py` | **NEW** | Emit `frontend_fix_handoff.md` + `codex_frontend_fix_prompt.md`; cite memory-read provenance; instruct fixing the **target frontend repo**, advisory only. | Modify target code; emit anything outside `qa-context/handoff/`; bypass `paths.py`. |
| `erpqa/reporters/module_report.py` | **NEW** | Render the PM-readable module reports from contracts + compare output; cite memory read. | Re-run comparison; introduce nondeterminism; write outside `qa-context/`. |
| `erpqa/generators/sql.py`, `erpqa/generators/handoff.py`, `erpqa/reporters/report.py`, `erpqa/adapters/intake.py` | v0.1, unchanged | — | Be modified by v0.2. |

> The **verification path** = `extractors → contracts → compare`. It is entirely LLM-free, offline, and deterministic. LLMs/skills appear only on the PROPOSE side (drafting policy/memory/contracts) and on the APPROVE side is the human.

## 3. Extractor Subsystem

### 3.1 Common interface

Every adapter (required, optional, or future) conforms to one interface so the registry can treat them uniformly. Illustrative signature only:

```
class Extractor(Protocol):
    name: str
    supported_extensions: frozenset[str]      # e.g. {".xlsx"}
    def available(self) -> bool: ...          # True iff its backing dep is importable
    def extract(self, path: Path) -> RawExtraction: ...   # READ-ONLY, OFFLINE

# RawExtraction = adapter-neutral payload + intrinsic provenance hint
@dataclass(frozen=True)
class RawExtraction:
    source_path: str
    kind: str                  # "tabular" | "text" | "machine" | "frontend" | "backend" | "procedure" | "image_manifest"
    records: list[dict]        # adapter-neutral rows/blocks/nodes
    provenance_hint: str       # adapter's max confidence ceiling ("high"|"medium"|"low")
    notes: list[str]
```

### 3.2 Registry (extension → adapter, double-gated)

```
def select(path, policy) -> Extractor | None:
    ext = path.suffix.lower()
    if ext not in policy.allowed_file_formats:   # gate 1: policy
        return None
    for adapter in _REGISTERED:                  # registration order = precedence
        if ext in adapter.supported_extensions and adapter.available():   # gate 2: availability
            return adapter
    return None
```

- **REQUIRED adapters** register unconditionally (their deps — stdlib, PyYAML, openpyxl — are always present).
- **OPTIONAL adapters** (`pandas` tabular path, `markitdown_adapter`) register only inside a guarded `try: import …` block; if the import fails, `available()` returns `False` and the registry simply skips them — the kit still runs on PyYAML + openpyxl alone (graceful degradation).
- **FUTURE adapters** (docling, unstructured) are **NOT registered** at all — documented in `docs/v0.2_EXTRACTOR_STRATEGY.md` and §13 here, never importable.

### 3.3 Flow: raw file → contract

```
   raw file              adapter                RawExtraction           normalizer            *_contract.yaml
 ┌──────────┐   select() ┌──────────────┐  extract() ┌────────────┐  normalize ┌──────────────────────────┐
 │ .xlsx    │──policy───▶│ xlsx_text     │──────────▶│ kind=tabular│──+confidence▶│ screen_contract.yaml      │
 │ .csv     │ +avail.    │ csv_adapter   │           │ records=[…] │  rules      │   fields[].source         │
 │ .md/.json│            │ markdown/json │           │ prov_hint   │             │   fields[].confidence     │
 │ frontend │            │ frontend_stat.│           └────────────┘             │   fields[].needs_human_…  │
 │ backend  │            │ backend_static│                                       └──────────────────────────┘
 │ .sql proc│            │ procedure_sql │   (write via paths.py containment guard)
 └──────────┘            └──────────────┘
```

### 3.4 Normalization → confidence rules (locked)

The normalizer turns `RawExtraction` into contract fields and stamps the provenance mixin. Ceilings (also in `docs/v0.2_PLAN.md` §5):

| Source kind | Default confidence | `needs_human_confirmation` | Ceiling |
|-------------|--------------------|----------------------------|---------|
| Explicit machine-readable (json/yaml) | high | false (if explicit) | high |
| Tabular/text (xlsx text, csv, markdown) | medium | true until confirmed | high |
| Static heuristic (frontend/backend/procedure, inferred mappings) | medium/low | true for inferred | medium |
| **Image-derived / OCR / MarkItDown-from-image** | **low** | **ALWAYS true** | **medium — NEVER high** |

`xlsx_image_manifest_adapter` lists embedded images + cell positions only (**no OCR**); its output is always low/medium + `needs_human_confirmation: true`.

## 4. Policy & Memory Subsystem

### 4.1 Load order & precedence

```
 read order (per command, BEFORE any work):
   1. project_policy.yaml ───┐ merge: module overrides/refines project
   2. module_policy.yaml  ───┴──▶ effective Policy
   3. project_memory.md   ───┐ refine: module refines/overrides project
   4. module_memory.md    ───┴──▶ effective Memory  + MemoryReadRecord(provenance)
```

| Layer | File | Precedence |
|-------|------|------------|
| Project policy | `project_policy.yaml` | base |
| Module policy | `modules/{MODULE}/module_policy.yaml` | **overrides/merges** project keys |
| Project memory | `project_memory.md` | base context |
| Module memory | `modules/{MODULE}/module_memory.md` | **refines/overrides** project memory |
| Assumptions | `project_assumptions.yaml` | human-confirmable; surfaced, never auto-applied |

### 4.2 Entry hook — "read policy + memory before work"

Every subcommand (granular and orchestrator) calls one shared entry hook **first**:

```
def load_context(project_path, module=None) -> RunContext:
    policy = policy.load_effective(project_path, module)     # may halt/draft if missing
    memory, mem_record = memory.load_effective(project_path, module)
    return RunContext(policy=policy, memory=memory, memory_read=mem_record)
```

`RunContext.memory_read` (a "memory was read" record: which files, when) is threaded into every report and handoff so outputs can cite it.

### 4.3 Missing `project_policy.yaml`

Behavior is policy-defined and never silent: either **halt** with a `UsageError` directing the user to run `policy-init`, **or** `policy-init` writes a **draft policy with `needs_human_confirmation: true`** that downstream commands refuse to act on until confirmed. Defaults are never silently assumed. Full schema/behavior: `docs/v0.2_PROJECT_POLICY_SPEC.md`.

## 5. Contract IR

The four contracts are the **stable intermediate representation** that `compare.py` and the exporters read. They share a provenance mixin:

```
ProvenanceMixin:  source: str       # adapter name + source_path
                  confidence: "high"|"medium"|"low"
                  needs_human_confirmation: bool
```

| Contract | Role | Key field sets (each field carries the provenance mixin) |
|----------|------|-----------------------------------------------------------|
| `screen_contract.yaml` | **Source of truth** (screen spec + backend evidence) | screen id/title; visible labels; column headers; search/filter fields; form fields; required/optional flags |
| `frontend_contract.yaml` | **Under test** (what the frontend implements) | rendered labels; rendered columns; rendered filters; form inputs; bound endpoints |
| `backend_contract.yaml` | Evidence (medium/low) | endpoints; request/response field names; entity/column names; validation rules (inferred) |
| `procedure_contract.yaml` | Deferred-priority evidence | procedure name; referenced tables/columns; output column set; business rules (text-parsed, **no exec**) |

Confidence values are constrained to v0.1's `CONFIDENCE_VALUES`; severities (assigned later by compare) to `SEVERITY_VALUES`. The IR is schema-validated by `contracts.py`; it contains **no comparison or extraction logic** — it is data only.

## 6. Comparison Engine (`core/compare.py`)

Deterministic, offline, LLM-free. Full mismatch taxonomy + severity table live in `docs/v0.2_SPEC.md`; sketch:

```
def compare(screen, frontend, backend, policy) -> CompareResult:
    truth = align(screen, backend)         # screen is authoritative; backend = corroborating evidence
    mismatches = []
    for field in truth.fields:
        fe = frontend.lookup(field.key)
        if fe is None:                      mismatches += MissingInFrontend(field)
        elif differs(field, fe):            mismatches += ValueMismatch(field, fe)
    for fe in frontend.extra_fields(truth): mismatches += ExtraInFrontend(fe)
    for m in mismatches:
        m.severity = assign_severity(m, policy)              # BLOCKER/MAJOR/MINOR
        m.needs_human_confirmation = (field.needs_human_confirmation or fe?.needs_human_confirmation)
        if policy.frontend_override_allowed is False and m.is_override_shaped:
            m.kind = "FRONTEND_OVERRIDE_FORBIDDEN"           # REPORT, never resolve
    return CompareResult(mismatches, memory_cited=ctx.memory_read)
```

Typed mismatches emitted (full list; defined in `docs/v0.2_SPEC.md`): **MissingInFrontend**, **ExtraInFrontend**, **LabelMismatch**, **ColumnMismatch**, **FilterFieldMismatch**, **FormFieldMismatch**, **RequiredFlagMismatch**, **EndpointBindingMismatch**, **FrontendOverrideForbidden**.

Rules: (1) the screen contract is always source of truth, backend only corroborates; (2) `needs_human_confirmation` propagates — any low-confidence side makes the mismatch require confirmation; (3) `frontend_override_allowed=false` is **enforced by reporting** the override as a mismatch, never by silently letting the frontend win; (4) same inputs → identical output (deterministic ordering).

## 7. Reporting & Handoff Subsystem

```
 CompareResult + contracts + RunContext.memory_read
        │
        ├─ reporters/module_report.py ──▶ qa-context/modules/{M}/reports/
        │     screen_frontend_mismatch_report.md, column_mismatch_report.md,
        │     frontend_contract_report.md, procedure_mapping_report.md
        │
        └─ generators/frontend_handoff.py ──▶ qa-context/modules/{M}/handoff/
              frontend_fix_handoff.md, codex_frontend_fix_prompt.md
```

| Output | MUST | MUST NOT |
|--------|------|----------|
| Module reports | Enumerate every detected mismatch with severity; **cite that memory was read** (from `memory_read`); mark `needs_human_confirmation` items. | Re-derive comparisons; auto-resolve conflicts. |
| `frontend_fix_handoff.md` | Be **advisory**; instruct fixing the **target frontend repository**, not the kit; reference the mismatch report. | Contain code edits to apply to the kit; modify target code. |
| `codex_frontend_fix_prompt.md` | Provide a Codex-ready prompt scoped to the target frontend repo, carrying memory + policy context. | Instruct DB execution or screen-spec/backend override. |

All writes route through `paths.py` / `module_paths.py` (containment guard). Handoffs are the only artifacts intended for humans/Codex to act on outside the kit — and even then, advisory.

## 8. CLI Integration

New subcommands slot into the **existing** `build_parser()` `commands` dict. Two shapes:

| Shape | Commands | Parser wiring |
|-------|----------|---------------|
| Project-level | `policy-init` | positional `project_path` only (same as v0.1 commands). |
| Module-level | `module-init`, `extract-spec`, `extract-frontend`, `extract-backend`, `extract-procedure`, `compare-contract`, `module-report`, `module-handoff`, `module-audit` | positional `project_path` **+ `--module <NAME>`**. |

Illustrative extension (additive — preserves the v0.1 loop):

```
commands = {
    # … v0.1 entries unchanged …
    "policy-init":      ("Init project policy/memory/assumptions", cmd_policy_init,      LEVEL_PROJECT),
    "module-init":      ("Scaffold a module",                       cmd_module_init,     LEVEL_MODULE),
    "extract-spec":     ("Spec → screen_contract.yaml",             cmd_extract_spec,    LEVEL_MODULE),
    "extract-frontend": ("Frontend → frontend_contract.yaml",       cmd_extract_frontend,LEVEL_MODULE),
    "extract-backend":  ("Backend → backend_contract.yaml",         cmd_extract_backend, LEVEL_MODULE),
    "extract-procedure":("Procedure → procedure_contract.yaml",     cmd_extract_proc,    LEVEL_MODULE),
    "compare-contract": ("Frontend vs source-of-truth",             cmd_compare,         LEVEL_MODULE),
    "module-report":    ("Render module reports",                   cmd_module_report,   LEVEL_MODULE),
    "module-handoff":   ("Emit frontend fix handoff",               cmd_module_handoff,  LEVEL_MODULE),
    "module-audit":     ("Orchestrate all module stages",           cmd_module_audit,    LEVEL_MODULE),
}
for name, (help_text, func, level) in commands.items():
    sub = subparsers.add_parser(name, help=help_text, description=help_text)
    sub.add_argument("project_path", help="Path to the target ERP project folder")
    if level == LEVEL_MODULE:
        sub.add_argument("--module", required=True, help="Module name")
    sub.set_defaults(func=func)
```

Every `cmd_*` calls `load_context()` (§4.2) first. **`module-audit`** is pure orchestration — it composes the granular stage functions in `policy.validation_order`, honoring `defer_until_frontend_verified` (procedure extraction/validation is skipped while unresolved frontend mismatches remain):

```
def cmd_module_audit(args):
    ctx = load_context(args.project_path, args.module)
    for stage in ctx.policy.validation_order:        # e.g. spec, frontend, backend, compare, report, handoff
        if stage == "procedure" and ctx.policy.defer_until_frontend_verified and frontend_unresolved():
            continue                                  # defer, do not run
        STAGE_FUNCS[stage](args, ctx)                 # reuse the same granular stage functions
```

## 11. Determinism, Safety & Boundaries

- **Verification path is deterministic & offline:** `extractors → contracts → compare`. No LLM call, no network, no DB.
- **No DB execution:** `procedure_sql_adapter` parses SQL text only; v0.1's SELECT-only `sql_safety.py` remains the gate for generated SQL.
- **No target-code modification:** extractors read; handoffs advise. The kit edits nothing outside `qa-context/`.
- **Module-scope, not full-repo:** runs target an extracted module folder; `module_manifest.yaml` records source paths; multi-repo (3 repos incl. ORD) mapping is policy-driven.
- **Write containment everywhere:** every writer routes through `paths.py`/`module_paths.py`; `assert_within_qa_context` is never bypassed.
- **Frontend override forbidden:** `frontend_override_allowed=false` enforced by *reporting* overrides as mismatches.
- **Provenance everywhere:** every contract field carries `source`/`confidence`/`needs_human_confirmation`; reports/handoffs cite memory reads.
